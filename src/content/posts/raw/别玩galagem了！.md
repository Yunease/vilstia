# 1.项目概述

## 1.1项目名称

【暂定名】：别玩galgame了！ | 别玩旮旯给木了！



## 1.2核心定位

【玩法类别】：avg视觉小说

【题材定位】：欢乐向，吐槽。

【用户定位】：是人就行，偏向都市搞笑，可以适当考虑添加一个“吵架”玩法，类音游，用renpy图形库实现。（14-28岁，70%男性受众+30%女性受众）

【价格定位】：demo和单线免费，完整版+后日谈约6~12块。

【上架平台】：itch（免费 单人线），steam（完整版+后日谈）。只有itch测试反响不错或者人气较好才会考虑发到steam，否则就是作着图一乐的。免费单人线先攻略诗怜吧，剩下两个按钮给一个锁住的按钮。

【产品定位】：剧情细腻，人设立体，画面多元但美术精度不高的小品级galgame，弱化恋爱元素，强调团队互相配合，其实也不算群像吧，神人云集这一块。



## 1.3短期项目目标

- 第一阶段：在一个月内实现剧情大纲并确认部分细节。
- 第二阶段：去画加寻找美术，找到画风合适，性价比优秀的画师。
- 第三阶段：做出demo，简单去B站进行宣发，把demo放在itch进行免费测试。
- 第四阶段：不众筹！直接埋头开做。
- 第五阶段：流程做完后进一步去B站宣发，同时去小红书，galgame笑话吧，国产galgame吧，小黑盒（注册一个），微博（注册一个），宣发图文帖，自己剪辑视频发宣发。
- 第六阶段：单线版发布至itch和taptap，试探市场反应，成功就制作完整版上steam，如果反响不错可以众筹卖周边。
- 第七阶段：我们的旮旯给木（哭泣）究竟会变成什么样子......（抽噎）很多时候我们听不清真正玩家的声音。

### 1.4.1开发周期 

只有我一个人做，外加几个朋友负责测试，部分美术会选择外包，背景图片会选择公用素材。实在不行自己建模自己渲染得了。

- 策划期：两个月。确认美术风格，整理需求和设计原型（byd还需要整理吗），写出剧情大纲（约10w字），找到合适的引擎（不用选了直接renpy）
- 开发期：八个月。写出完整剧情，搭建系统框架，完善ui。在第二个月做出可玩demo，发送给朋友测试。
- 优化期：两个月。体验优化，测试测试bug，反正没有配音随便搞。
- 发布期：准备好后全平台同步发布。

# 2.游戏核心设定

## 2.1世界观设定

不是我说白了旮旯给木能有啥世界观啊，偏三次元，虽然很想选在广东奈何我没去过，还是放在自家家乡西安吧，起码这里熟一点，还能顺带推销一下自己的母校。

三个可攻略角色是线下参与开发，三个不可攻略角色是线上远程开发，省点美术钱。

线下开发的三个角色+主角是核心成员，几个不可攻略角色没有立绘，用仿微信聊天框的ui，只绘制小头像即可。成本可以砍至不剩1500。



## 2.2角色设定

### 2.2.1主角设定

- 身份：刚开始玩galgame的大学生，对galgame充满了好奇，虽然有些忐忑，但是很乐意和志趣相投的人分享自己的爱好。
- 性格：性格虽然内向，但很乐于交朋友，对工作充满信心。但是不太会和同事打交道，有一点生涩。~~不是我说白了galgame的主角还能有啥性格啊，没给你弄成摄像头就不错了。~~
- 自定义：允许玩家自定义名字和选择语气倾向对话（不影响流程走向）。



### 2.2.2可攻略角色设定

可攻略角色的剧情基本根据真实事情改编而来，但为了混淆（别搞得别人一眼看出来了），这里引入`附加设定`这一概念，附加设定为把其他人的抽象事安在这个角色身上。

此外，大部分`标准设定`都是内部发生的事情，外部并不知情，外界知情的如炒作吃黑流量等行为，这里为了避嫌还是不写了（可惜了）



此外，身高主要是给约稿画师参考的，实际游戏中应该不会太提到，我本意想按照男生平均身高-10≈女性平均身高，但是老板高198，就算-10也高的吓人，于是含泪推出了神人-20后抹零算法。（那我不就只剩140了吗。。。）

# 4.系统设定

## 4.1单人线的好感度设定

由于游戏初期只有一个角色线可以选择，因此上述的玩法相当于就是单纯在玩，并不打算真正添加好感度系统。因此也不需要设置好感度系统。



## 4.2正式版的好感度设定

实际上影响好感度的选项，应该用单独样式的选项标记出现（会提醒进入战斗）

现阶段规划应该有8次选项，那么最平均分布情况应该是3+3+2。

- 好感度数值范围[0,8]
- 好感度只增不减，虽然可以增加一些减好感度的动画，但是为了一致性实际上是+0好感度。
- 好感度每次都只+1，防止数据溢出。
- 部分对话和好感度有关，但不会太多，主要填充细节。
- 进入个人线最低的好感度要求为4，这样就避免出现超过最低要求还同好感度的判定问题了，因为我懒得写优先级。

### 4.4.2收集系统提示和UI

菜单中未解锁的内容，应该提示大致的解锁要求，引导玩家能够顺利解锁。

解锁新内容的时候，应该播放一段音效，并以弹幕的形式在右上角提示玩家收集新内容（音乐就不用收集了）

- **未解锁内容提示**
  - 收集菜单中，未解锁项不直接剧透，仅显示简明引导文字，如 “完成某角色支线解锁”“达成好感度条件解锁”“触发隐藏场景解锁”。

- **解锁即时反馈**

  - 玩家在游戏中解锁新收集品时：

    1. 播放一段短促清脆的解锁提示音效（同样是8bit风格的）；

    1. 屏幕右上角以弹幕形式弹出文字，如 “已解锁新 CG”“发现新彩蛋” 等，停留数秒后自动消失。

- **其他规则**
  - 收集进度与存档绑定，跨存档进度独立；出现新的音乐时，不弹出弹窗（解绑）但会收集，专注于视觉类内容。



## 4.5 配音系统

由于成本有限，目前配音系统采用类undertale的电流音方案。

思路非常简单，但是不确定是否可行，实在不行砍掉配音也可以。



### 4.5.1原始音频的获取

本系统所有配音音频均为8bit 电子音色，通过免费工具生成，无版权风险。

- 生成工具：Bfxr / BeepBox
- 音频规格：短音效（0.05~0.15 秒）、WAV 格式、无杂音、适配文字播放节奏
- 音频类型：
  1. **通用基础音**：干净的方波 / 脉冲电流音，用于日常对话
  2. **情绪特色音**：带音调变化、波形差异，对应生气、开心、低落等情绪



### 4.5.2角色音色存储和特殊语气

- 每个角色拥有独立配音文件夹，实现音色完全区分
- 文件夹结构：`audio/voice/角色名/`
- 每个角色文件夹内包含：
  - 20 个基础随机音频（`1.wav ~ 20.wav`）：用于日常对话，随机播放避免听觉疲劳
  - 5 个固定情绪音频：用于特殊台词，强化剧情表现力

系统内置 5 种固定情绪配音，可在剧本中手动标记触发，仅对指定单句台词生效：

1. `angry`（愤怒 / 呵斥）
2. `happy`（开心 / 兴奋）
3. `sad`（难过 / 低落）
4. `soft`（温柔 / 轻声）
5. `loud`（高亢 / 大喊）（等）



### 4.5.3简要实现

这里只写出简单的逻辑，具体的思路在”技术实现“中详细讲解。



#### 4.5.3.1核心逻辑

1. **默认播放**：角色对话时，每显示一个文字，自动随机播放该角色的 1~20 号基础音频
2. **情绪触发**：在特殊台词前添加一行标记，当前整句对话自动切换为指定情绪配音
3. **自动恢复**：情绪台词播放完毕后，下一句自动切回默认随机音
4. **低侵入性**：不修改核心剧情结构，无配音时可直接关闭系统，无逻辑残留



#### 4.5.3.2运行流程

1. 定义角色 → 绑定对应配音文件夹
2. 日常对话 → 自动随机播放基础音
3. 关键剧情台词 → 手动标记情绪 → 播放情绪音
4. 恢复日常对话 → 继续随机播放



# 5.美术设计

## 5.1 美术风格定位

因为比较穷，为了尽可能带来更好的美术体验，我打算尝试多元美术风格混搭。

主要美术风格如下：

- 像素风格：毕竟自己会画，这是最简单的，直接用在角色小头像和部分小表情上，也可以绘制一些贴图。
- 仿柚子社风格：版本轮椅，谁能拒绝一个柚子社的小表情包，而且也不贵
- 纯色模糊渐变：用于底部台词栏，经典省经费操作，反正别人也干了。
- 仿蔚蓝档案的干净画风风格：便宜而且相对来说还可以，服饰不用太复杂，用于立绘。
- 实拍滤镜风格：用于背景，免费素材很多
- 三渲二：用于cg，用blender实现打光来做出相对来说还可以接受的cg画面，导出后可以用简单的sai2笔刷再改改。



但是，为了解决画风混乱的问题，同一个画面不能同时存在多个风格混搭，背景图片尽量选择和立绘一样色彩干净，光影关系简单的图。
