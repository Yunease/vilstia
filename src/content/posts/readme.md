# 1.项目文件夹管理结构

```json
./
├── dream/
├── gallery/
│	└── star/
├── message/
│	└── 特殊规则/
├── photo/
│	└── star/
├── poetry/
│	├── 中学时/
│	└── 俳句/
├── raw/
├── 游戏开发/
├── 段子/
├── 技术文章/
│	└── 开源软件推荐系列/
├── 轻小说/
├── 如星空般深蓝/
├── 水蒸气蛋糕/
├── 小厨时间/
├── 同人文/
│	└── 心之轨迹/
│		└── 特殊规则/
├── 学习笔记/
├── 杂文/
│	└── 实习日志/
└── readme.md
```

接下来我将依次讲解每个文件夹目录要如何使用。

## dream

这个文件夹存放所有记录下来的梦，并且tag和category必须为dream

这里的文章会展示在剪影的 `叙梦协定` 板块中。



## gallery

这个文件夹存放个人画作，其中必须要有image字段作为title大图，正文需要留空。

其中，二级文件夹 `star` 是星标图片，含义为个人满意的画作，star图需要在tag中写下star。



## message

这个文件夹的文件太多，拥有特殊规则：

`message.md` 等文件，tag和category必须为mess。所有mess文件都不会显示在除了 `心灵碎片` 的任何地方。

mess文件按照月份储存，如：

```json
.
├── 2026_1/
├── 2026_2/
├── 2026_3/
├── xxxx_x/
└── old/
```

old为所有2025年及以前的mess文件，其内部有更完整的文件夹目录，不作展示。未来如果考古出远古mess，请放在old文件夹的对应位置中。

其余文件，应该按照日期放入 `xxxx_x` 的文件夹中，代表年月，。

mess文件的创建方法，请找到本文件夹中的 `mess.md` 文件，对其ctrl cv就会自动排列出数字标的 `mess - 副本（xx）.md` ，相对方便。

> 如果在直接在手机端github上提交文件，可以命名为mess-phone-x.md



mess文件拥有特殊的字段： `mood`

```json
mood字段可选:
{
    焦躁: "#E06C75",
    消沉: "#7F8C8D",
    怨恨: "#C792EA",
    开心: "#3CB371",
    平和: "#FFB6C1",
    振奋: "#F4A460"
}
```

`mood` 字段会对应一个固定的十六进制颜色，在 `心里碎片` 页面中，需要展示 `mood` 信息，请必须填写。



## photo

除了tag和category需要改为 `photo` 外和 `gallery` 规则完全一样。



## poetry

这个文件夹储存个人诗作，并拥有如下结构：

```json
poetry/
├── 俳句/
├── 歌曲/
└── 中学时/
```

`poetry` 文件夹中的所有文件，其 `category` 都为 诗与歌，而 `tag` 则需要看类别，在歌曲文件夹中的为 歌，在其他文件夹中的为 诗。

其中， `中学时` 是已经完结的个人诗篇汇总。



## raw

这个文件夹存放的是暂未写完的文章，任何文章都可以在这里新建，但最终写完需要移走。

请参考 `Astro对md文件的规范要求` ，注意默认的字段必须填写。



## 段子

这里存放一些抽象幽默的玩笑。

所有段子文件的 `tag` 都应该至少包含 段子，所有 `category` 都应该是 下里巴人。 

如果某一个段子文件符合其他tag，也可以添加。



## 技术文章

这个文件夹里会存放一些技术性的文章。注意！技术文章一定不包含游戏开发的内容，游戏开发内容的文章需要写进游戏开发文件夹中，也不能包括学习笔记，技术文章主要内容为解决问题的办法和开源推荐。

技术文章的一定包含 `博客文章` 这个 `tag` ，其余可以自行添加新的 `tag` 



## 轻小说

这个文件夹存放自己写的轻小说。这个文件夹目录中的 `tag` 和 `categroy` 都是 `轻小说` 

三题故事也会放入这个文件夹，不过不需要专门标注//未来可能会修改。



## 如星空般深蓝

这个文件夹是一个正在写的散文集，其 `tag` 只能是 `随笔` ， `category` 只能是 `如星空般深蓝`



## 水蒸气蛋糕

这个文件夹是另一个正在写的散文集，其规则和 `如星空般深蓝` 相似，不做赘述。



## 同人文

这个文件夹是存放我写的轻小说和oc小说的目录。其他同人文正常保存即可，心之轨迹拥有特殊的规则：

1.` category ` 是心之轨迹

2.` tag ` 是 宝可梦，同人文

3.拥有特殊的文件夹路径

4.终极文件夹中有一个py.py，是之前写的一个文档转换工具，未来可能会需要再次使用。



其余md文件的tag规则为 对应ip，对应cp（如果有）的同人文



## 学习笔记

这个文件夹存放我的学习笔记，例如 ` English ` ，  ` JavaScript ` 。

学习笔记的 ` tag ` 为对应的领域，如 ` CSS ` ，`category` 为 `学习`

如果未来要学习新的领域，新建新的文件夹目录即可。



## 其他规则

> 复合tag规则：
>
> ​	当一篇文章出现多个tag时，在中括号内用英文逗号+空格隔开，例如：
>
> ​	[日常, 学习]



> Astro对md文件的规范要求：
>
> ​	本框架要求md文档必须含有两个字段： ` title ` `  published`
>
> ​	任何md文件，只要在post目录下，都必须包含这两个字段。
