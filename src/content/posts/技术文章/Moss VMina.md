# 项目简介

Moss VMina 是一个探索「个人化人工智能」的开源项目。

它尝试通过收集个人长期积累的文本资料（博客、文章、日记、通讯软件的聊天记录等），经过数据清洗、增强以及大语言模型微调，使 AI 学习特定用户的表达习惯、知识结构与交流方式。

与传统的 AI 助手不同，Moss VMina 并不追求成为一个拥有最大参数量的通用模型，也不追求强大的 Coding 能力，而是探索：

> 如果一个人的文字、思想和表达方式能够被 AI 学习，是否可以创造一个更加贴近个人的智能助手？

目前，Moss VMina 已实现：

- 基于个人语料训练的人格化 LLM
- 本地大语言模型推理
- SFT 微调训练流程
- 本地 TTS 语音输出
- LLM 与 TTS 联动
- OpenAI API 兼容接口
- QQ 等通讯平台接入

本项目希望降低个人开发者探索 AI 的门槛，让更多人能够尝试训练属于自己的模型。

## 环境要求

### 基础环境

推荐： 

- Python >= 3.10 
- CUDA >= 12.x 
- Git - Git LFS  



### 硬件要求推理：

#### 推理：

最低配置（本人实际测试环境）：

- 内存：16GB
- 显存：6GB

推荐配置：

- 内存：16GB+
- 显存：8GB+



#### 训练：

最低配置（本人实际测试环境）：

- 内存：16GB
- 显存：6GB

推荐配置：

- 内存：16GB+
- 显存：12GB+ 甚至 24GB



具体的硬件需求取决于模型大小和你的训练策略，以及数据集本身，不同模型大小、batch size 和训练方式会显著影响显存需求。尽管 RTX 3060( 6 GB ) 勉强可以跑，但更推荐你使用性能更加出色的 GPU。在我的本地训练中，4b左右的模型几乎是最低配置的极限，即便训练完成，后续推理时的时延也有些无法接受——如果只是用来简短聊天，以目前水平来看，4b的模型几乎是极限了。

如果你的显存足够，在后续的训练阶段可以不必使用那么激进的压缩优化策略。

如果显存不足，也可以使用：

- 云端 GPU 算力
- ~~把项目发给研究生朋友的空闲电脑~~



## 安装依赖

克隆项目：

```bash
git clone https://github.com/Yunease/Moss-VMina.git
cd Moss-VMina
```



由于模型文件、密钥以及第三方服务体积较大，你克隆好的项目是不能直接使用的，因为：

- .env文件被忽略了（意味着你无法运行反向推理获取 Prompt 的脚本，但仍然可以参考思路实现，或自己补上 api key ）
- 缺少大模型本体
- 缺少 Lora 权重
- 缺少 GPT-Sovits 的API（如果要使用 TTS 服务，需要下载 GPT-Sovits 并运行 api_v2.py ）
