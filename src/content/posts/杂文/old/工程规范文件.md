# AVG 剧本编写规范（简化版）

## 🎯 目标

本规范用于：

- 编写视觉小说（AVG）剧本
- 保证**写作流畅**
- 保证**可程序解析**
- 支持后期自动转换为 CSV / Ren’Py 脚本

------

# 一、基本结构

## 1. 场景

```md
# scene: 场景ID
```

示例：

```md
# scene: quarrel_01
```

------

## 2. 标签（跳转目标）

```md
# label: 标签ID
```

示例：

```md
# label: after_fight
```

------

# 二、对话

## 基本格式

```md
角色: 台词
```

示例：

```md
A: 你为什么又迟到了？
B: 我已经解释过很多次了。
```

------

## 关键词（用于战斗）

```md
A: 你这个【废物】。
```

### 金色关键词：

```md
A: 你这个【!废物】。
```

规则：

- `【】` → 普通关键词
- `【!】` → 金色关键词

------

# 三、指令系统

## 基本规则

```md
@指令 参数
```

一行一个指令。

------

## 常用指令

### 场景控制

```md
@bg 背景名
@music 音乐名
```

------

### 战斗控制

```md
@battle start
@battle end
```

------

### 条件分支

```md
@if win
@if lose
@if fc
```

说明：

- 表示战斗结果分支
- 后面缩进内容属于该分支

示例：

```md
@if win
    B: ……你这次说得太狠了。

@if lose
    B: 果然，你还是不行。
```

------

### 夜晚事件（表现奖励）

```md
@night fc
```

示例：

```md
@night fc
B: ……其实我有点被你说动了。
```

------

### 效果

```md
@shake
@sound 音效名
```

------

# 四、选择分支

```md
@choice

- 选项文本:
    jump: 标签ID
```

示例：

```md
@choice

- 继续争吵:
    jump: argue_more

- 冷静下来:
    jump: calm_down
```

------

# 五、完整示例

```md
# scene: quarrel_01

@bg classroom
@music tense

A: 你为什么又迟到了？

B: 我已经说过很多次了。

@battle start

A: 你这个【废物】。

@shake
@sound attack

@if win
    B: ……你这次说得太狠了。

@if lose
    B: 果然，你还是不行。

@battle end

@choice

- 继续争吵:
    jump: argue_more

- 冷静下来:
    jump: calm_down

# label: argue_more

A: 你根本没有反省。

# label: calm_down

A: ……算了。
```

------

# 六、解析约定（给程序用）

每一行解析为一个结构：

| 类型      | 示例         |
| --------- | ------------ |
| scene     | `# scene:`   |
| label     | `# label:`   |
| say       | `角色: 台词` |
| command   | `@xxx`       |
| choice    | `- 选项`     |
| jump      | `jump:`      |
| condition | `@if`        |

------

# 七、设计原则（必须遵守）

## 1️⃣ 一行只做一件事

✔ 正确：

```md
A: 你这个【废物】。
@shake
```

❌ 错误：

```md
A: 你这个【废物】 @shake
```

------

## 2️⃣ 保持可读性

剧本必须：

- 可以像小说一样阅读
- 不依赖工具理解结构

------

## 3️⃣ 不提前复杂化

当前版本不支持：

- 数值计算
- 多层嵌套逻辑
- 复杂变量系统

------

# 八、目标

该规范应满足：

- ✔ 可直接写完整剧本
- ✔ 可自动转换为 CSV
- ✔ 可生成 Ren’Py 脚本
- ✔ 不影响写作体验
